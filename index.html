Project Overview

Pressure monitoring for safety-critical aerospace conditions.

This project models a real-time pressure monitoring system for an aerospace or pilot-support environment. The ESP32 reads pressure from a BMP180 sensor, checks whether the pressure has dropped below a critical threshold, and turns on a red alert LED when the condition becomes unsafe.

The key requirement is timing: the pressure detection to alert LED path must finish within a 20 ms hard deadline. The system uses FreeRTOS tasks, an interrupt, a queue, and a semaphore to keep deadline-critical behavior predictable.

RTOS Design

Task and deadline summary.

Task / ISR Priority Timing Role Criticality
buttonISR Interrupt Aperiodic Detects physical button press Soft
heartbeat_task 1 1000 ms on / 1000 ms off Shows system alive with green LED Soft
bmp_reading 2 5 ms period Reads pressure and detects unsafe condition Hard
sensor_logger_task 1 Queue-driven Stores latest sensor value for display Soft
event_handler_task 2 ~10 ms loop Handles pressure alerts and LED response Hard
web_server_task 3 ~2 ms loop Handles web control requests Soft
01

Synchronization

The sensor task sends pressure readings through a queue, button/web actions signal through a binary semaphore, and shared output is protected to avoid unsafe concurrent access.

02

Determinism Proof

The Wokwi logic analyzer was used to measure the path from pressure threshold detection to the alert LED response. Normal testing showed approximately 10 ms response time.

03

Design Tradeoff

Non-critical display and logging behavior is separated from the hard pressure alert path so visual safety feedback remains responsive under load.
